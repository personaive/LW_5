## Задача 11

### Текст задачи

Дано натуральное число n. Дописать к нему цифру k в конец и в начало. 

### Алгоритм решения

1\. Начало программы, устанавливаем кодировку UTF-8.

2\. Определяем предикат digits\_count(N, 1) для однозначного числа.

3\. Если число ≥ 10, делим его на 10 и рекурсивно считаем цифры.

4\. Увеличиваем счётчик на 1 при возвращении из рекурсии.

5\. Определяем append\_end(N, K, Result) для приписывания цифры в конец: умножаем N на 10 и прибавляем K.

6\. Определяем append\_start(N, K, Result) для приписывания цифры в начало.

7\. Считаем количество цифр числа через digits\_count.

8\. Вычисляем степень 10, равную количеству цифр.

9\. Приписываем цифру K в начало: умножаем K на 10^Count и прибавляем N.

10\. Определяем главный предикат go для взаимодействия с пользователем.

11\. Запрашиваем ввод натурального числа N.

12\. Запрашиваем ввод цифры K (0–9).

13\. Проверяем корректность ввода: N > 0, 0 ≤ K ≤ 9.

14\. Если ввод корректный, вычисляем число с K в конце через append\_end.

15\. Вычисляем число с K в начале через append\_start.

16\. Выводим результат с цифрой в конце.

17\. Выводим результат с цифрой в начале.

18\. Если ввод некорректный, выводим сообщение об ошибке.

19\. Завершаем выполнение предиката go.

20\. Автоматически запускаем go при загрузке файла.

## Задача 11

### Текст задачи

Между какими степенями двойки расположены все положительные элементы

списка?

### Алгоритм решения

1\. Начало программы, устанавливаем кодировку UTF-8.

2\. Определяем предикат min\_max\_pos(\[H|T], Min, Max) для поиска минимума и максимума неотрицательных элементов.

3\. Если первый элемент H ≥ 0, инициализируем CurMin = H, CurMax = H, и продолжаем обработку хвоста T рекурсивно.

4\. Определяем предикат min\_max\_pos(\[H|T], CurMin, CurMax, Min, Max) для рекурсивной обработки списка.

5\. Если H ≥ 0, обновляем текущие минимум и максимум: NewMin = min(CurMin, H), NewMax = max(CurMax, H).

6\. Если H < 0, оставляем CurMin и CurMax без изменений.

7\. Рекурсивно вызываем min\_max\_pos для хвоста списка с обновлёнными Min и Max.

8\. Определяем базовый случай min\_max\_pos(\[], Min, Max, Min, Max) — список пуст, возвращаем текущие Min и Max.

9\. Определяем предикат power\_down(N, P) для нахождения нижней степени двойки.

10\. Специальный случай: если N = 0, нижняя степень = 0.

11\. Если 2^K ≤ N < 2^(K+1), возвращаем K как нижнюю степень двойки.

12\. Иначе увеличиваем K на 1 и рекурсивно вызываем power\_down(N, K1, P).

13\. Определяем предикат power\_up(N, P) для нахождения верхней степени двойки.

14\. Специальный случай: если N = 0, верхняя степень = 0.

15\. Если 2^K ≥ N, возвращаем K как верхнюю степень двойки.

16\. Иначе увеличиваем K на 1 и рекурсивно вызываем power\_up(N, K1, P).

17\. Определяем главный предикат go для взаимодействия с пользователем.

18\. Запрашиваем ввод списка List у пользователя.

19\. Вызываем min\_max\_pos(List, Min, Max) для нахождения минимума и максимума неотрицательных элементов.

20\. Вызываем power\_down(Min, A) и power\_up(Max, B) для нахождения степеней двойки.

21\. Выводим результат: элементы лежат между 2^A и 2^B.

22\. Если нет подходящих элементов, выводим сообщение об ошибке.

23\. Завершаем выполнение предиката go.

24\. Автоматически запускаем go при загрузке файла.

## Задача 11

### Текст задачи

Реализовать сортировку списка выбором. 

### Алгоритм решения

1\. Начало программы, устанавливаем кодировку UTF-8.

2\. Определяем предикат min\_elem(\[X], X) — базовый случай для поиска минимума в односоставном списке.

3\. Определяем min\_elem(\[H|T], Min) для поиска минимума в списке с несколькими элементами.

4\. Рекурсивно находим минимум MinT в хвосте списка T.

5\. Сравниваем текущий элемент H с MinT: если H < MinT, Min = H, иначе Min = MinT.

6\. Определяем предикат remove\_elem(X, \[X|T], T) для удаления первого найденного элемента X из списка.

7\. Определяем remove\_elem(X, \[H|T], \[H|R]) для рекурсивного удаления элемента из хвоста списка.

8\. Определяем базовый случай сортировки selection\_sort(\[], \[]) — пустой список остаётся пустым.

9\. Определяем selection\_sort(List, \[Min|SortedTail]) для сортировки непустого списка.

10\. Находим минимум Min в списке List с помощью min\_elem.

11\. Удаляем первый найденный минимум из List с помощью remove\_elem, получаем Rest.

12\. Рекурсивно сортируем остаток списка Rest, получаем SortedTail.

13\. Объединяем Min и SortedTail в отсортированный список.

14\. Определяем главный предикат go для взаимодействия с пользователем.

15\. Запрашиваем ввод списка List у пользователя.

16\. Вызываем selection\_sort(List, Sorted) для сортировки списка.

17\. Выводим результат: "Отсортированный список:" и содержимое Sorted.

18\. Завершаем выполнение предиката go.

19\. Автоматически запускаем go при загрузке файла.

## Задача 11

### Текст задачи

Трое ребят вышли гулять с собакой, кошкой и хомячком. Известно, что Петя не

любит кошек и живет в одном подъезде с хозяйкой хомячка. Лена дружит с

Таней, гуляющей с кошкой. Определить, с каким животным гулял каждый из

детей?

### Алгоритм решения

1\. Начало программы, устанавливаем кодировку UTF-8.

2\. Определяем главный предикат go для решения задачи.

3\. Задаём список всех животных: Animals = \[dog, cat, hamster].

4\. Используем permutation(\[PetAnimal, LenaAnimal, TanyaAnimal], Animals) для перебора всех возможных распределений животных между детьми.

5\. Накладываем условие: PetAnimal \\= cat (Петя не любит кошек).

6\. Накладываем условие: TanyaAnimal = cat (Таня гуляет с кошкой).

7\. Накладываем условие: PetAnimal \\= hamster (Петя не с хомячком, живёт с хозяйкой хомячка).

8\. Выводим результат для Пети: format('Петя гулял с \~w\~n', \[PetAnimal]).

9\. Выводим результат для Лены: format('Лена гуляла с \~w\~n', \[LenaAnimal]).

10\. Выводим результат для Тани: format('Таня гуляла с \~w\~n', \[TanyaAnimal]).

11\. Завершаем выполнение предиката go.

12\. Автоматически запускаем go при загрузке файла с помощью :- go.
